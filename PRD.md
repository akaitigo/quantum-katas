# PRD: quantum-katas

## 概要

量子コンピューティングの基礎概念をインタラクティブなコーディング演習（カタ形式）で学べるWebアプリケーション。各カタには量子ゲートの動作原理の図解と、穴埋め形式のCirqコードが含まれ、ブラウザ上で実行結果を即座に確認できる。

## 背景・動機

量子コンピューティングは急速に発展しているが、入門の敷居が高い。数学的背景（線形代数・複素数）の知識が前提とされることが多く、プログラマが「手を動かしながら学ぶ」環境が不足している。本プロジェクトは「穴埋め → 実行 → 結果確認」のサイクルを数秒で完結させ、量子力学の数学的背景なしでも直感的に量子ゲートを習得できる段階的カリキュラムを提供する。

## ターゲットユーザー

- プログラミング経験はあるが量子コンピューティング未経験のエンジニア
- CS学生で量子計算の入門を探している人
- 量子コンピューティングに興味があるが数学的ハードルを感じている人

## MVP機能

### F1: 10段階の入門カタ

段階的に量子コンピューティングの概念を学ぶカリキュラム:

| # | カタ名 | 学習概念 |
|---|--------|---------|
| 1 | Hello Qubit | 量子ビットの基本、|0> と |1> 状態 |
| 2 | NOT Gate (X Gate) | パウリXゲート、ビット反転 |
| 3 | Superposition | アダマールゲート、重ね合わせ状態 |
| 4 | Measurement | 測定と確率、量子状態の崩壊 |
| 5 | Phase Kick | Zゲート、位相変化 |
| 6 | Entanglement | CNOTゲート、量子もつれ |
| 7 | Bell States | ベル状態の4パターン |
| 8 | Quantum Teleportation | 量子テレポーテーションプロトコル |
| 9 | Deutsch's Algorithm | 最初の量子アルゴリズム |
| 10 | Grover's Search | グローバー探索アルゴリズム入門 |

各カタの構成:
- **解説セクション**: 概念の図解（Bloch球、回路図）
- **穴埋めコード**: Cirqを使った量子回路コード（キー部分が空欄）
- **実行ボタン**: ブラウザ上でシミュレーション実行
- **結果表示**: 測定結果のヒストグラム・状態ベクトル表示

### F2: ブラウザ内Pythonエディタと量子回路シミュレーション実行

- Monaco Editorベースのコードエディタ
- Python (Cirq) コードをバックエンドで実行
- 実行結果（測定ヒストグラム、状態ベクトル）をリアルタイム表示
- シンタックスハイライト、自動補完

### F3: 正解判定とヒント機能付き進捗トラッカー

- 各カタの期待出力と実際の出力を比較して正解判定
- 3段階のヒントシステム（ヒント1: 方向性、ヒント2: 具体的なゲート名、ヒント3: ほぼ答え）
- 進捗バー表示（完了カタ数 / 全カタ数）
- ローカルストレージで進捗を永続化

```
Frontend (React + Monaco Editor)
    |
    | REST API (JSON)
    v
Backend (FastAPI)
    |
    | Python execution
    v
Cirq Simulator (量子回路シミュレーション)
```

## 非機能要件

- **レスポンス**: コード実行は3秒以内に結果返却
- **セキュリティ**: サンドボックス実行（任意コード実行の制限）
- **可用性**: 静的コンテンツはCDN配信可能な構成

## スコープ外（v1）

- ユーザー認証・アカウント管理
- サーバーサイド進捗保存
- カスタムカタ作成機能
- 実機量子コンピュータへの接続
- マルチ言語（Qiskit等）対応

## 受け入れ条件

- [ ] 10個のカタが全て実行可能
- [ ] 穴埋めコードを正しく埋めると正解判定される
- [ ] ヒント機能が各カタで動作する
- [ ] 進捗がブラウザリロード後も保持される
- [ ] コード実行結果がヒストグラムとして表示される

## 競合・先行事例

| サービス | 差別化ポイント |
|---------|--------------|
| Microsoft Quantum Katas | Q#ベース（Cirqではない）、IDE必要 |
| IBM Quantum Learning | Qiskitベース、カタ形式ではない |
| Qiskit Textbook | テキストブック形式、インタラクティブ性が限定的 |

本プロジェクトの差別化: **Cirqベース + ブラウザ完結 + 穴埋めカタ形式**の組み合わせ。
